## Data Integrations

### Repliers API

The primary data source powering Portal. The frontend never calls Repliers directly вҖ” it talks to a proxy backend at `NEXT_PUBLIC_API_URL` (default `http://localhost:8080`, with an `/api` prefix), which forwards requests to Repliers. The endpoints below are reached through that proxy.

| Endpoint         | Purpose                        |
| ---------------- | ------------------------------ |
| `/api/listings`  | Property search and details    |
| `/api/locations` | Geography data (cities, areas) |
| `/api/estimate`  | AVM property valuations        |
| `/api/users`     | Authentication and profiles    |
| `/api/favorites` | Saved properties               |
| `/api/searches`  | Saved search queries           |

### Content Management

Portal supports multiple content sources via three CMS clients, with per-route resolution configured in `@configs/cms`:

| Source    | Use Case                       |
| --------- | ------------------------------ |
| WordPress | Blog, dynamic pages            |
| Ghost     | Blog, dynamic pages            |
| Markdown  | CMS-served Markdown content    |
| Local MDX | Static pages, forms (fallback) |

> **Full documentation:** [Content Management Guide](/product-guide/content-management)

## Authentication

### Supported Methods

Authentication is a server/proxy redirect flow: the backend returns an `AuthResponse.url` that the client redirects to. The `AuthProvider` values are `google`, `facebook`, and `otp`.

| Method                        | Provider   |
| ----------------------------- | ---------- |
| Email OTP (one-time passcode) | `otp`      |
| Google OAuth                  | `google`   |
| Facebook OAuth                | `facebook` |
