# Glossary

Terms that are **not obvious from context** — real-estate / MLS jargon and Portal-specific
concepts. Self-evident words (bedrooms, mortgage, map view…) are intentionally omitted; the
user-facing features are described in the [Product Guide](/product-guide).

## Real estate & MLS

- **MLS** — Multiple Listing Service. The broker-shared listing database the Portal ultimately
  sources from (through the proxy backend, never directly).
- **IDX** — Internet Data Exchange. The public feed brokers may display on their sites — the
  reduced dataset shown to anonymous visitors.
- **VOW** — Virtual Office Website. A registered-user context that exposes **fuller** listing
  data (history, sold detail) than IDX. In config this is a separate board id — see
  **`vowBoardId`** below.
- **RESO** — Real Estate Standards Organization. The data-standard some boards follow; a tenant
  may ship a separate RESO variant of its build.
- **DOM / CDOM** — Days on Market / **Cumulative** Days on Market. CDOM keeps counting across
  relists, so it can exceed DOM for a re-listed property.
- **Off-Market** — not publicly listed but potentially available.
- **Pocket Listing** — sold privately without MLS exposure.
- **HOA** — Homeowners Association; recurring fees for shared amenities (surfaced on condo/PDP).
- **Zoning** — the legal land-use classification that limits how a lot may be used.
- **MSA** — Metropolitan Statistical Area; a metro region centered on an urban core.
- **Walk Score / Transit Score** — third-party 0–100 walkability / transit-access ratings.
- **Flood Zone** — FEMA-designated flood-risk area.

## Listing status

Status is derived from two Repliers fields — `status` (`A` active / `U` unavailable) and
`lastStatus` (the granular lifecycle code) — and grouped into **sale / rent / sold / inactive**
by `utils/listings/status.ts`.

- **`lastStatus` codes** — `New`, `Pc` (price change), `Ext` (extension), `Dft` (deal fell
  through) are on-market; `Sc` / `Sce` (sold conditionally, ± escape clause) and `Lc` / `Lce`
  (leased conditionally) are conditional; `Sld` (sold) / `Lsd` (leased) are firm; `Sus`
  (suspended), `Exp` (expired), `Ter` (terminated) are off-market.
- **"Sold" is broader than a firm sale.** The sold group is defined as _everything not active_ —
  so it covers firm `Sld` / `Lsd`, **conditional (pending) deals**, off-market listings, and (for
  guests) restricted ones.
- **Pending = conditional sale/lease** (`Sc` / `Sce` / `Lc` / `Lce`): the deal is accepted but
  **not yet firm**, yet it is **labelled and grouped as "Sold"**. A "Sold" badge therefore does
  not always mean the transaction has closed.
- **Restricted → Sold for guests.** A VOW-gated listing a signed-out user isn't entitled to see is
  resolved to the sold group (`resolveStatusGroup`), hiding its live data behind the sold treatment.

## Valuation & market

- **AVM** — Automated Valuation Model. The algorithm behind the [Estimate](/product-guide/estimate)
  feature; not an appraisal.
- **CMA** — Comparative Market Analysis. An agent-prepared pricing report (the human counterpart
  to an AVM).
- **Comps** — comparable recently-sold nearby properties used to derive a value.
- **Assessment vs Appraisal** — Assessment is the government's tax valuation; Appraisal is a
  licensed professional's market-value opinion. Neither equals an AVM estimate.
- **Absorption Rate / Months of Supply** — how fast inventory sells / how long current inventory
  would last at the current pace. Drive the market-temperature (buyer's vs seller's) gauge.
- **YoY / MoM** — year-over-year / month-over-month comparison, used in market reports.

## Portal concepts

- **Tenant** — a build-time brand/instance selected by `NEXT_PUBLIC_APP_CONFIGURATION`; each has
  its own `configs/<tenant>/` overrides. Everything branded is resolved per tenant at build time.
- **`defaults` tenant** — the base/demo build (deployed to portal.repliers.com). It is the
  fallback for every tenant and the default build — **not** the separate `repliers` tenant.
- **Config override** — files under `configs/<tenant>/` that shadow `configs/defaults/` via
  path aliases (`@configs/*` resolves **tenant-first, then defaults**). Only differing files exist.
- **Component override** — a `_<tenant>/` folder next to a component that a webpack plugin swaps
  in for that tenant only. The second, independent override mechanism alongside config overrides.
- **`app` (GrowthBook)** — the id feature flags are scoped by, one per tenant. The default
  build's `app` is `defaults` (not `repliers`) — relevant when force-enabling default-only flags.
- **Feature flag** — a per-tenant capability toggle (`@configs/features`). Can be static
  (committed `features.ts`) or synced from GrowthBook via `generate:features`.
- **`defaultBoardId` / `vowBoardId` / `singleBoard`** — MLS board routing. When the two ids are
  equal the tenant is single-board (`singleBoard: true`) and the board id is omitted from URLs
  and API calls; a multi-board (IDX + VOW) tenant sets them apart and `singleBoard: false`.
- **Scrubbed** — a field whose value is masked with the `!scrubbed!` sentinel because the current
  user isn't entitled to it (VOW/registration gating). The `scrubbed()` util detects it before render.
- **`source`** — the location-data source for a tenant (usually `'MLS'`); required for every
  non-default tenant in `location.ts`. Location ids are namespaced by source.
- **LiveBy** — the external data source powering map **overlays** (neighborhoods, postal codes,
  school districts, schools).
- **Overlay** — an extra, config-driven map data layer. Polygons render on Mapbox **GL**; point
  markers render as **DOM** markers (see [Map Overlays](/technical-guide/map-overlays)).
- **Proxy backend** — the backend the frontend calls; it proxies the Repliers MLS API and owns
  the OAuth handshake / JWT. The frontend never calls Repliers (or an OAuth provider) directly.
- **OTP** — the passwordless auth mechanism: an email/phone one-time code, gated by the
  `messaging` flag (see [Authentication](/technical-guide/authentication)).
- **AI Agent (EVAS)** — an **external** AI embedded via iframe + `postMessage`; swappable and
  distinct from the built-in, Repliers-powered [AI Chat](/product-guide/ai-chat).
- **Widget** — a reusable component embeddable in CMS-managed pages.
- **Transformer** — a function that maps raw API responses to the frontend's data shape.
- **Buildings (inverted WP naming)** — in the WordPress/ACF integration the CPT **"buildings"** is
  the _group/container_, while the **"categories"** taxonomy holds the _individual buildings_ — the
  naming is inverted from intuition.
