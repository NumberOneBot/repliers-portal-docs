# Site Anatomy

A technical fast-navigation map of the site. Each row is a user-facing section; the columns
deep-link into that section's **How it works** page — providers, component tree, and config.
Use this when you know _what part of the app_ you're touching and want its internals.

<Callout type="info">
  The functional "what & why" of each section lives in the [Product
  Guide](/product-guide); the technical anatomy lives on each section's **How it
  works** sub-page. This page ties them together. Global state (app-wide
  providers) is in [State Management](/technical-guide/state-management).
</Callout>

## Global providers

Every section renders inside the app-wide provider tree (`app/_providers.tsx`):

```
NextIntl → MUI cache → Theme → UserProvider → DialogProvider → SnackbarProvider
  → [FavoritesProvider?] → [SaveSearchProvider?] → [ImageFavoritesProvider?]
```

The last three are feature-gated at build time. Section-specific ("scoped") providers mount
inside their own routes — see each section below and
[State Management](/technical-guide/state-management).
