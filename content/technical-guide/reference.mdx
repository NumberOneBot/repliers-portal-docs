# Reference

Quick lookup tables for common values, codes, and configurations.

## Listing Statuses

The MLS `status` field is coarse — only two values. Granular states (sold,
expired, leased, price change, …) live in the separate `lastStatus` field as
3-character codes.

### `status`

| Code | Meaning     | Description                               |
| ---- | ----------- | ----------------------------------------- |
| `A`  | Active      | On-market / available                     |
| `U`  | Unavailable | Off-market (sold, expired, terminated, …) |

### `lastStatus`

3-character codes mapped to display labels in
`configs/defaults/listings.ts` (`statusLabels`).

| Code  | Label                                   |
| ----- | --------------------------------------- |
| `Sus` | Suspended                               |
| `Exp` | Expired                                 |
| `Sld` | Sold                                    |
| `Ter` | Terminated                              |
| `Dft` | Deal Fell Through                       |
| `Lsd` | Leased                                  |
| `Sc`  | Sold Conditionally                      |
| `Sce` | Sold Conditionally with Escape Clause   |
| `Lc`  | Leased Conditionally                    |
| `Lce` | Leased Conditionally with Escape Clause |
| `Pc`  | Price Change                            |
| `Ext` | Extension                               |
| `New` | New                                     |

### Usage

```typescript
// Coarse availability
if (listing.status === 'A') {
  // Show "Active" badge
}

// Granular state via lastStatus
const soldListings = listings.filter((l) => l.lastStatus === 'Sld')
```

## Property Types

| Code           | Type          | Description                 |
| -------------- | ------------- | --------------------------- |
| `residential`  | Residential   | Single-family detached home |
| `condo`        | Condo         | Condominium unit            |
| `townhome`     | Townhome      | Attached multi-floor home   |
| `semiDetached` | Semi-Detached | Duplex, shared wall         |
| `multiFamily`  | Multi-Family  | Multiple units (investment) |
| `land`         | Land          | Vacant land                 |
| `commercial`   | Commercial    | Commercial property         |

### Usage

```typescript
// Filter by type
const condos = listings.filter((l) => l.propertyType === 'condo')

// Multiple types
const residential = listings.filter((l) =>
  ['residential', 'townhome', 'semiDetached'].includes(l.propertyType)
)
```

## Sort Options

| Code              | Description    | Direction     |
| ----------------- | -------------- | ------------- |
| `listPriceAsc`    | Price          | Low to High   |
| `listPriceDesc`   | Price          | High to Low   |
| `soldDateDesc`    | Sold Date      | Newest first  |
| `updatedOnDesc`   | Updated        | Most recent   |
| `bedsDesc`        | Bedrooms       | Most first    |
| `sqftDesc`        | Square Feet    | Largest first |
| `daysOnMarketAsc` | Days on Market | Newest first  |

### Usage

```typescript
// Set sort
setFilters({ ...filters, sortBy: 'listPriceDesc' })
```

## Feature Flags

Static per-tenant flags from `configs/<tenant>/features.ts` (the `defaults`
build ships 22 flags). The full list with descriptions, defaults, dependencies,
and how flags are set lives in
[Capabilities & Flags](/getting-started/capabilities-and-flags). Defaults below
are the `configs/defaults/features.ts` values.

| Flag                     | Description                         | Default |
| ------------------------ | ----------------------------------- | ------- |
| `map`                    | Interactive map search              | `true`  |
| `estimate`               | Home valuation / AVM                | `true`  |
| `dashboard`              | Market dashboard                    | `true`  |
| `blog`                   | Blog section (CMS)                  | `true`  |
| `locations`              | Locations directory                 | `false` |
| `buildings`              | Buildings directory                 | `false` |
| `openHouse`              | Open houses                         | `false` |
| `favorites`              | Save favorite listings              | `true`  |
| `saveSearch`             | Save search filters                 | `true`  |
| `liveBy`                 | LiveBy demographics / overlays      | `true`  |
| `aiSearch`               | Natural-language query parsing      | `true`  |
| `aiChat`                 | Conversational AI chat in search    | `true`  |
| `aiAgent`                | AI agent (embedded EVAS iframe)     | `false` |
| `blurRestrictedProperty` | Blur restricted listings for guests | `false` |

### Usage

```typescript
import features from '@configs/features'

// Page-level gate
if (!features.blog) return <Page40XTemplate />

// Conditional render
{features.aiChat && <AIChatWidget />}
```

## URL Parameters

### Search Filters

| Parameter      | Type   | Example                  |
| -------------- | ------ | ------------------------ |
| `city`         | string | `Seattle`                |
| `area`         | string | `King County`            |
| `neighborhood` | string | `Belltown`               |
| `status`       | enum   | `active`, `sold`, `rent` |
| `type`         | string | `condo`, `residential`   |
| `minPrice`     | number | `500000`                 |
| `maxPrice`     | number | `1000000`                |
| `minBeds`      | number | `2`                      |
| `maxBeds`      | number | `4`                      |
| `minBaths`     | number | `1`                      |
| `minSqft`      | number | `1000`                   |
| `maxSqft`      | number | `3000`                   |
| `minYearBuilt` | number | `2000`                   |
| `soldWithin`   | number | `90` (days)              |
| `sortBy`       | enum   | `listPriceDesc`          |
| `page`         | number | `1`                      |

### Example URL

```
/search/map?city=Seattle&status=active&minPrice=500000&maxPrice=1000000&minBeds=2&type=condo&sortBy=listPriceDesc
```

## Board IDs

The MLS feed is selected by a numeric `boardId`, set per tenant in `configs/<tenant>/search.ts`
(`defaultBoardId` = IDX feed, `vowBoardId` = VOW feed; multi-board tenants differ between the
two). Known board ids and the MLS / region they map to:

| Board id | MLS         | Region                                |
| -------- | ----------- | ------------------------------------- |
| `2`      | TRREB       | Toronto / GTA, ON                     |
| `18`     | CREB        | Calgary, AB                           |
| `68`     | MLS PIN     | Boston, MA                            |
| `71`     | SmartMLS    | Connecticut, USA                      |
| `90`     | TRREB (VOW) | Toronto / GTA, ON                     |
| `91`     | TRREB (IDX) | Toronto / GTA, ON                     |
| `110`    | —           | sample / demo data (`defaults` build) |
| `149`    | NWMLS       | Seattle, WA                           |

## Breakpoints

Custom values from `configs/defaults/theme/breakpoints.ts` — they **override the MUI defaults**
(600/900/1200/1536):

| Name | Width   | Usage         |
| ---- | ------- | ------------- |
| `xs` | 0px+    | Mobile        |
| `sm` | 640px+  | Tablet        |
| `md` | 960px+  | Small desktop |
| `lg` | 1280px+ | Desktop       |
| `xl` | 1920px+ | Large desktop |

### Usage

```tsx
import { useMediaQuery, useTheme } from '@mui/material'

const theme = useTheme()
const isMobile = useMediaQuery(theme.breakpoints.down('sm'))
```
