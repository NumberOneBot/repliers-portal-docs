# Portal Technical Guide

**Version:** 1.1
**Last Updated:** July 2026
**Audience:** Frontend Developers

## Overview

Portal is a Next.js frontend application that provides property search websites for real estate professionals. The frontend never calls the Repliers MLS API directly: it talks to a proxy backend (`NEXT_PUBLIC_API_URL`), which forwards requests to the Repliers API. It supports multiple client deployments through a multi-tenant architecture.

## Tech Stack

| Category            | Technology                                   |
| ------------------- | -------------------------------------------- |
| **Framework**       | Next.js 16 (App Router)                      |
| **Language**        | TypeScript                                   |
| **UI Library**      | MUI v7 (Material UI)                         |
| **Styling**         | Emotion (CSS-in-JS)                          |
| **Maps**            | Mapbox GL, Mapbox GL Draw                    |
| **Forms**           | React Hook Form + Joi validation             |
| **i18n**            | next-intl                                    |
| **State**           | React Context (Providers)                    |
| **Node**            | 22+ (the frontend pins engines.node to 24.x) |
| **Package Manager** | pnpm 10+                                     |

## Key Concepts

| Concept              | Description                                       |
| -------------------- | ------------------------------------------------- |
| **Multi-tenant**     | One codebase serves multiple branded client sites |
| **Override Pattern** | Tenant-specific files override defaults           |
| **Providers**        | React Context for global state management         |
| **API Services**     | Modules for Repliers API communication            |
| **Feature Flags**    | Toggle features per tenant                        |
